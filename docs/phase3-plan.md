# Phase 3：多维分析增强

> 创建日期：2026-04-29 | 状态：设计完成，Phase 2 完成后实施

## 0. 前置条件

Phase 2 已完成：
- 用户体系 + 会员付费在线运行
- 每日数据/模型自动更新
- 技术面预测稳定运行

---

## 1. 目标

将单维度的技术面预测升级为多维度的智能投研平台，引入基本面因子和舆情因子提升模型准确率，增加回测验证策略有效性，拓展微信小程序端。

---

## 2. 功能模块

### 2.1 基本面数据接入

| 数据维度 | 具体指标 | 数据源 |
|---------|---------|--------|
| 估值指标 | PE / PB / PS / PEG | akshare 财报接口 |
| 盈利能力 | ROE / ROA / 毛利率 / 净利率 | akshare 财务指标 |
| 成长性 | 营收增长率 / 净利润增长率（同比/环比） | akshare 利润表 |
| 偿债能力 | 资产负债率 / 流动比率 | akshare 资产负债表 |
| 分红 | 股息率 / 分红比例 | akshare 分红数据 |

**技术方案**：
- 新建 `financial_data` 表存储季度财报数据
- 季度更新即可（财报频率决定）
- 作为模型附加特征输入（20 维 → 30+ 维）

### 2.2 舆情分析

| 数据维度 | 数据源 | 更新频率 |
|---------|--------|---------|
| 新闻情感 | 个股相关新闻标题/摘要 → NLP 情感打分 | 每日 |
| 北向资金 | 沪深港通每日流向 | 每日 |
| 龙虎榜 | 每日龙虎榜上榜个股+营业部 | 每日 |
| 融资融券 | 融资余额/融券余量变化 | 每日 |
| 机构研报 | 券商研报评级+目标价 | 每周 |

**NLP 技术方案**：

```
新闻爬取 → 文本清洗 → 情感分析
                         ├── 方式 A：SnowNLP（轻量，中文友好）
                         ├── 方式 B：transformers 中文 FinBERT
                         └── 方式 C：调用大模型 API（更准确但需成本）
```

初期推荐 **SnowNLP** 快速验证，效果不够再升级到 FinBERT。

**新建表**：
```sql
news_sentiment (id, ts_code, news_date, title, source, sentiment_score, summary)
north_flow     (id, trade_date, net_inflow, top_stocks JSONB)
dragon_tiger   (id, trade_date, ts_code, reason, buy_amount, sell_amount, net_amount)
margin_trade   (id, trade_date, ts_code, margin_balance, short_balance)
```

### 2.3 回测引擎

| 功能 | 说明 |
|------|------|
| 策略定义 | 基于模型预测方向 + 置信度阈值，生成买卖信号 |
| 历史回测 | 在历史区间模拟交易，计算收益率/夏普比率/最大回撤 |
| 多策略对比 | 同一时间段跑多个策略，横向对比绩效 |
| 绩效归因 | 收益拆分：行业贡献 / 择时贡献 / 选股贡献 |
| 可视化报告 | 净值曲线 / 回撤曲线 / 月度收益热力图 |

**技术方案**：参考 `backtrader` 的设计，自建轻量回测引擎（避免引入重型依赖）：

```
回测输入: 每日推荐列表 + 置信度
       ↓
模拟交易: 初始资金 100 万，等权买入 TOP-N，持 N 天换仓
       ↓
绩效指标: 累计收益、年化收益、夏普比率、最大回撤、胜率
       ↓
输出报告: JSON + ECharts 可视化
```

### 2.4 微信小程序

| 页面 | 功能 |
|------|------|
| 首页 | 今日推荐 TOP-5 + 大盘概览 |
| 搜个股 | 股票搜索 + 详情（K 线 + 预测 + 基本面 + 舆情） |
| 盯盘 | 自选列表 + 实时提醒 |
| 我的 | 会员信息 + 设置 |

小程序技术栈：
- 框架：`uni-app`（一套代码编译 H5 + 微信小程序）
- 或原生微信小程序 + WXS

如果前端用的是 Vue.js，`uni-app` 迁移成本最低。

### 2.5 模型升级

| 升级方向 | 方案 |
|---------|------|
| 更多模型 | 加入 Informer / Autoformer（长序列预测 SOTA） |
| 特征融合 | 技术面 + 基本面 + 舆情 → 多模态特征 |
| 集成学习 | 多模型加权投票，提升稳定性 |
| 增量训练 | 每日新增数据不重训全量，finetune 最后一层 |
| 不确定性估计 | MC Dropout 输出预测置信区间 |

## 4. 容器化 & 云部署

```yaml
# docker-compose.yml 目标结构
services:
  nginx:        # 反向代理 + 静态资源
  api:          # FastAPI 主服务
  scheduler:    # 定时任务（数据更新/训练）
  sentiment:    # 舆情分析 worker（异步任务队列）
  db:           # PostgreSQL 16
  redis:        # 缓存 + 消息队列
  # 可选 GPU worker（模型训练）
```

部署目标：
- **初期**：单台云服务器 4C8G（阿里云 / 腾讯云）
- **后期**：考虑 GPU 实例用于模型训练加速
