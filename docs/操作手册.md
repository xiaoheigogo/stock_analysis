## 1. 环境准备

### 1.1 系统要求

| 组件 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.10+ | 实际使用 3.12.3 |
| PostgreSQL | 12+ | 数据存储 |
| pip | 22+ | 依赖安装 |
| 磁盘空间 | 5GB+ | 模型和数据存储 |

### 1.2 安装 PostgreSQL

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

**创建数据库:**
```bash
sudo -u postgres psql
```

在 psql 中执行:
```sql
CREATE DATABASE stock;
-- 如果密码不是 123456，请对应修改 config.yaml
ALTER USER postgres PASSWORD '123456';
\q
```

### 1.3 创建虚拟环境并安装依赖

```bash
# 进入项目目录
cd /home/xiaohei/pyproject/stock_analysis

# 创建虚拟环境
python3.12 -m venv venv38

# 激活虚拟环境
source venv38/bin/activate

# 安装基础依赖
pip install -r requirements.txt

# 安装 PyTorch（CPU 版本）
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu

# 或安装 CUDA 版本（如有 NVIDIA GPU）
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

> 本机当前: Python 3.12.3, PyTorch 2.11.0+cu130, CUDA 可用。

**验证安装:**
```bash
python -c "import torch; print(f'PyTorch {torch.__version__}, CUDA: {torch.cuda.is_available()}')"
```

### 1.4 配置 config.yaml

项目根目录下的 `config.yaml` 是主配置文件。核心配置项如下:

```yaml
database:
  postgres:
    host: "localhost"      # 数据库地址
    port: 5432             # 数据库端口
    database: "stock"      # 数据库名
    user: "postgres"       # 用户名
    password: "123456"     # 密码（生产环境请修改）

proxy:
  enabled: true            # 是否启用代理（数据获取时使用）
  http_proxy: "http://127.0.0.1:7897"
  https_proxy: "http://127.0.0.1:7897"
```

> 如果你的环境不需要代理，将 `proxy.enabled` 设为 `false`。

## 3. 数据获取

### 3.1 一键更新所有数据

```bash
python main.py update --all
```

这会依次更新: 交易日历 -> 股票列表 -> 指数K线 -> 股票K线（增量）。

### 3.2 分步更新

```bash
# 仅更新交易日历
python main.py update --trade-date

# 仅更新股票列表
python main.py update --stock-list

# 更新指定股票的K线数据（增量，推荐）
python main.py update --stock-kline --ts-codes 000001,000002,000004

# 全量刷新某只股票的所有历史数据
python main.py update --stock-kline --ts-codes 000001 --full-update
```

### 3.3 更新更多股票

A股6位代码，不足6位前面补0:

```bash
# 更新平安银行和万科A
python main.py update --stock-kline --ts-codes 000001,000002

# 更新大盘蓝筹（示例）
python main.py update --stock-kline --ts-codes 000001,000002,600519,601318
```

### 3.4 数据更新时间说明

- 交易日收盘后（约17:00）数据可用
- 增量更新仅获取本地缺失的日期
- 首次运行建议使用 `--full-update` 获取完整历史

## 4. 数据清洗

数据清洗包括: 补齐停牌日、计算技术指标（MA5/MA10/MACD/RSI14）、编码时间特征、归一化。

### 4.1 清洗指定股票

```bash
# 清洗单只股票，指定日期范围
python main.py clean --ts-codes 000001 --start-date 2020-01-01 --end-date 2026-05-16

# 同时清洗多只
python main.py clean --ts-codes 000001,000002,000004 --start-date 2020-01-01 --end-date 2026-05-16
```

### 4.2 清洗全部未清洗的股票

```bash
python main.py clean --all-stocks
```

此命令自动发现数据库中 `close_clean IS NULL` 的股票并逐只清洗。

### 4.3 清洗流程说明

1. 从数据库读取原始数据（`*_raw` 列）
2. 对齐交易日历，补齐停牌日（前向填充）
3. 计算技术指标: MA5、MA10、MACD DIF、RSI14
4. 编码时间特征: 月份/日期/星期 → sin/cos 循环编码
5. 滚动窗口 StandardScaler 归一化
6. 写入数据库 `*_clean` 列

归一化参数保存在 `data_process/scaler_X.pkl`，第一只股票清洗时生成，后续股票复用。

## 5. 模型训练

系统支持 4 种模型: LSTM、GRU、CNN-LSTM、Transformer（BiLSTM 是 LSTM 的双向变体）。

### 5.1 快速训练（推荐命令）

```bash
# 训练 LSTM 模型
python main.py train --model lstm --ts-codes 000001,000002,000004 --epochs 50 --output-dir ./output

# 训练 GRU 模型
python main.py train --model gru --ts-codes 000001,000002,000004 --epochs 50 --output-dir ./output

# 训练 CNN-LSTM 模型
python main.py train --model cnn_lstm --ts-codes 000001,000002,000004 --epochs 50 --output-dir ./output

# 训练 Transformer 模型
python main.py train --model transformer --ts-codes 000001,000002,000004 --epochs 50 --output-dir ./output
```

### 5.2 完整参数示例

```bash
python main.py train \
  --model lstm \
  --ts-codes 000001,000002,000004 \
  --seq-len 30 \
  --pred-len 10 \
  --epochs 50 \
  --batch-size 32 \
  --lr 0.001 \
  --loss mse \
  --optimizer adam \
  --scheduler reduce \
  --output-dir ./output
```

### 5.3 参数说明

| 参数 | 可选值 | 默认值 | 说明 |
|------|-------|--------|------|
| `--model` | lstm, gru, bilstm, cnn_lstm, transformer | lstm | 模型类型 |
| `--ts-codes` | 逗号分隔的股票代码 | - | 训练数据包含的股票 |
| `--seq-len` | 整数 | 30 | 输入序列长度（历史天数） |
| `--pred-len` | 整数 | 10 | 预测天数 |
| `--epochs` | 整数 | 50 | 训练轮数 |
| `--batch-size` | 整数 | 32 | 批次大小 |
| `--lr` | 浮点数 | 0.001 | 学习率 |
| `--loss` | mse, mae, huber | mse | 损失函数 |
| `--optimizer` | adam, sgd, adamw | adam | 优化器 |
| `--scheduler` | step, reduce, cosine, none | reduce | 学习率调度策略 |
| `--task_type` | regression, classification | regression | 任务类型 |
| `--output-dir` | 路径 | ./output | 模型和日志输出目录 |

### 5.4 训练输出

训练完成后，`./output/models/` 目录包含:

```
output/models/
├── best_model.pth          # 验证集上表现最好的模型（API 默认加载）
├── last_model.pth          # 最后一个 epoch 的模型
└── checkpoint_epoch_*.pth  # 周期性检查点
```

`output/logs/` 目录包含训练日志。

### 5.5 已有模型

项目已训练好一个 GRU 模型，位于 `output/models/best_model.pth`，在 000001/000002/000004 上训练完成，可直接用于预测。

## 7. 使用预测功能 (API)

### 7.1 API 端点一览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/` | API 根路径，列出所有端点 |
| POST | `/api/predict` | 预测单只股票 |
| GET | `/api/recommend` | 获取今日推荐 |
| GET | `/api/stock/{ts_code}` | 获取股票历史数据 |

### 7.2 健康检查

```bash
curl -s http://localhost:8000/api/health | python -m json.tool
```

响应字段:
- `status`: 服务状态（healthy）
- `database`: 数据库连接状态（connected / error）
- `model_available`: 模型是否已加载（true / false）

### 7.3 预测单只股票

```bash
curl -s -X POST http://localhost:8000/api/predict \
  -H "Content-Type: application/json" \
  -d '{"ts_code": "000001", "seq_len": 30, "pred_len": 10}' \
  | python -m json.tool
```

请求体:

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `ts_code` | string | 是 | - | 6位股票代码 |
| `seq_len` | int | 否 | 30 | 输入序列长度 |
| `pred_len` | int | 否 | 10 | 预测天数 |

响应示例:
```json
{
  "ts_code": "000001",
  "ts_name": "平安银行",
  "predicted_return": 0.0234,
  "confidence": 0.72,
  "risk_score": 0.35,
  "predicted_sequence": [12.34, 12.38, ...],
  "industry": "银行",
  "reasons": [
    "近期成交量放大"
  ]
}
```

### 7.4 获取今日推荐

```bash
# 默认参数（前10只，最大风险0.5，最低置信度0.6）
curl -s "http://localhost:8000/api/recommend" | python -m json.tool

# 自定义筛选条件
curl -s "http://localhost:8000/api/recommend?top_n=5&max_risk=0.4&min_confidence=0.7" | python -m json.tool
```

查询参数:

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `top_n` | int | 10 | 返回推荐数量（1-50） |
| `max_risk` | float | 0.5 | 最大风险阈值（0.0-1.0） |
| `min_confidence` | float | 0.6 | 最低置信度阈值（0.0-1.0） |

## 8. 查看前端

### 8.1 访问地址

服务启动后，浏览器打开: **http://localhost:8000**

### 8.2 页面功能

前端共 5 个页面，通过顶部导航栏切换:

| 页面 | 功能说明 |
|------|---------|
| **首页** | 系统简介，展示核心功能入口。左侧边栏显示数据库连接状态、模型加载状态、股票总数、最新数据日期。 |

## 9. 常见问题

### 9.1 数据库连接失败

**症状:** `psycopg2.OperationalError: could not connect to server`

**排查步骤:**
```bash
# 1. 检查 PostgreSQL 是否运行
sudo systemctl status postgresql

# 2. 如果未运行，启动它
sudo systemctl start postgresql

# 3. 测试连接
python -c "from utils.db_conn import postgres; print('连接成功')"
```

确认 `config.yaml` 中的数据库配置与本机 PostgreSQL 设置一致（host/port/database/user/password）。

### 9.2 数据获取失败

**症状:** akshare 调用超时或返回空数据

**解决方案:**
1. 检查网络连接，确保能访问外部网络
2. 确认代理设置: 如果 `config.yaml` 中 `proxy.enabled: true`，确保代理客户端（如 Clash）正在运行且端口一致（默认 7897）
3. 关闭代理: 将 `proxy.enabled` 设为 `false`
4. 降低请求频率，避免被数据源限流

### 9.3 模型文件不存在

**症状:** `FileNotFoundError` 或 `model_available: false`

**说明:** API 启动时自动加载 `output/models/best_model.pth`。如果该文件不存在，`model_available` 显示 `false`。

**解决方案:** 先训练模型（参考第5节），确保 `output/models/best_model.pth` 存在。

### 9.4 预测结果为空

**可能原因:**
1. 股票数据未清洗（`close_clean` 列为 NULL）：运行 `python main.py clean --ts-codes <代码>`
2. 股票不在 `stock_list` 表中：运行 `python main.py update --stock-list`
3. 数据量不足，无法构建序列（需要至少 seq_len + pred_len 条数据）

### 9.5 PyTorch 安装问题

**使用清华镜像加速:**
```bash
pip install torch torchvision -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**验证 PyTorch 安装:**
```bash
python -c "import torch; print(torch.__version__)"
```

### 9.6 内存不足

训练时内存不足的解决方案:
- 减少 `--batch-size`（例如改为 16 或 8）
- 减少 `--seq-len`（例如改为 20）
- 减少训练股票数量

### 9.7 查看日志

```bash
# 应用日志
tail -f logs/stock.log

# 训练日志
ls output/logs/

# 查看最近的训练历史
cat output/logs/*.json 2>/dev/null | tail -50
```

### 9.8 端口被占用

**症状:** `OSError: [Errno 98] Address already in use`

**解决方案:**
```bash
# 查找占用 8000 端口的进程
lsof -i :8000

# 终止该进程
kill -9 <PID>

# 或使用其他端口启动
uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload
```

### 9.9 虚拟环境激活确认

确保每条命令前已激活虚拟环境:

```bash
source venv38/bin/activate
which python  # 应显示 .../venv38/bin/python
```
