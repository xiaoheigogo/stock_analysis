# 股票预测系统需求文档

## 1. 项目概述

本项目旨在构建一个基于深度学习的股票预测系统，实现股价走势预测、自动选股和交易推荐功能。系统采用云端自动化部署，支持实时数据更新、多模型比较和交易接口集成。

## 2. 核心目标

### 2.1 主要预测目标
- **任务类型**: 回归任务
- **输入**: 历史30天K线数据 + 技术指标
- **输出**: 未来10天收盘价预测序列
- **模型**: LSTM/GRU等时序模型

### 2.2 关键特性
1. **实时预测系统**: 收盘后自动更新数据、训练模型、生成预测
2. **多模型比较框架**: 支持LSTM、GRU、Transformer等多种时序模型对比
3. **交易接口集成**: 集成券商API，支持模拟交易和实盘交易
4. **云端自动化部署**: 基于阿里云PAI平台的完整云端解决方案

## 3. 功能需求

### 3.1 数据管理模块
- [ ] **数据同步**: 每日自动从akshare API同步全市场股票数据
- [ ] **数据清洗**: 停牌日补全、异常值处理、特征工程
- [ ] **数据存储**: PostgreSQL存储原始和清洗后数据，OSS存储训练数据
- [ ] **数据版本管理**: 支持数据版本回溯和增量更新

### 3.2 模型训练模块
- [ ] **基础LSTM模型**: 实现单层LSTM(128) + Dropout(0.2) + FC(10)回归模型
- [ ] **多模型框架**: 支持LSTM、GRU、BiLSTM、CNN-LSTM、Transformer等模型
- [ ] **训练流水线**: 数据加载、模型训练、验证、保存一体化
- [ ] **超参数优化**: 支持网格搜索、随机搜索、贝叶斯优化
- [ ] **模型评估**: MSE、MAE、MAPE、IC（信息系数）等评估指标

### 3.3 预测服务模块
- [ ] **实时预测**: 收盘后自动运行预测，生成Top-N股票推荐
- [ ] **批量预测**: 支持全市场股票批量预测
- [ ] **API接口**: RESTful API提供预测服务
- [ ] **结果存储**: 预测结果存入数据库，支持历史查询

### 3.4 交易接口模块
- [ ] **模拟交易**: 基于预测结果的模拟交易回测
- [ ] **实盘接口**: 集成券商API（如华泰、东方财富等）
- [ ] **风控系统**: 仓位控制、止损止盈、风险监控
- [ ] **交易记录**: 完整记录所有交易操作和结果

### 3.5 监控告警模块
- [ ] **模型监控**: 模型性能衰减检测和告警
- [ ] **数据监控**: 数据质量检查和异常告警
- [ ] **系统监控**: 服务健康状态监控
- [ ] **业绩跟踪**: 预测准确率、收益率等关键指标跟踪

### 3.6 Web展示模块
- [ ] **股票可视化**: K线图、技术指标、预测曲线展示
- [ ] **预测结果**: Top-N推荐股票列表和详细分析
- [ ] **模型对比**: 不同模型性能对比展示
- [ ] **参数配置**: 在线调整模型参数和训练配置

## 7. 模型设计

### 7.1 基础LSTM模型
```python
class StockLSTM(nn.Module):
    def __init__(self, input_dim=20, hidden_dim=128, output_dim=10, dropout=0.2):
        super().__init__()
        self.lstm = nn.LSTM(input_dim, hidden_dim, batch_first=True)
        self.dropout = nn.Dropout(dropout)
        self.fc = nn.Linear(hidden_dim, output_dim)

    def forward(self, x):
        lstm_out, _ = self.lstm(x)
        last_out = lstm_out[:, -1, :]  # 取最后一个时间步
        out = self.dropout(last_out)
        out = self.fc(out)
        return out
```

### 7.2 模型扩展
1. **GRU模型**: 参数更少，训练更快
2. **BiLSTM模型**: 双向LSTM，捕捉前后依赖
3. **CNN-LSTM模型**: CNN提取空间特征，LSTM处理时序
4. **Transformer模型**: Self-Attention机制，捕捉长距离依赖

## 11. 风险评估和缓解措施

### 技术风险
1. **数据质量风险**: 数据源不稳定或数据错误
   - 缓解: 多数据源验证，数据质量检查
2. **模型过拟合风险**: 模型在训练集表现好，测试集差
   - 缓解: 正则化、交叉验证、早停法
3. **性能风险**: 全市场预测耗时过长
   - 缓解: 分布式预测，模型优化，缓存机制

### 业务风险
1. **预测准确性风险**: 预测结果不准确导致投资损失
   - 缓解: 模拟交易验证，风险提示，分散投资
2. **合规风险**: 金融数据使用和交易合规问题
   - 缓解: 法律咨询，合规审查，用户协议
3. **市场风险**: 市场异常波动影响模型效果
   - 缓解: 风险控制，止损机制，人工干预
